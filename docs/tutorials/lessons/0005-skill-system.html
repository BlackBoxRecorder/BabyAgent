Lesson 5: Skill 系统 — Agent 的领域知识注入

📅 课程 5 / 6 ⏱ 13 分钟 📂 src/skills.ts + .qoder/skills/
🎯 目标：理解 Skill 系统如何通过 markdown 文件给 agent 注入领域知识和行为指令——不写代码，只写 prompt。

1. 什么是 Skill？

一个 Skill 就是一个包含 YAML frontmatter + Markdown 指令 的 SKILL.md 文件，放在 skills/ 目录的子目录中。它告诉 LLM 如何完成特定任务。

Skill 系统的核心流程：

SKILL.md（文件）→ 读取 description → 拼入系统提示词
                        ↓
            LLM 发现匹配的任务 → 读取完整 SKILL.md
                        ↓
            SKILL.md 内容成为对话消息 → LLM 按指令执行
💡 项目当前有 20+ 个 skill！ 查看 .qoder/skills/ 目录，包含了 teach、tdd、code-review、prototype 等。

2. SKILL.md 文件格式

一个典型的 SKILL.md 文件：

--- name: teach description: Teach the user a new skill or concept... disable-model-invocation: true --- The user has asked you to teach them something...
字段 必填 说明
name 否 Skill 名，默认用目录名
description 是 LLM 看到的一句话描述，决定何时激活
disable-model-invocation 否 为 true 时不出现在系统提示词中，仅通过 /skill:name 命令触发

    if (idx !== -1) all[idx] = ps;  // 覆盖

  }
}

🔍 设计要点：

3.2 组装到系统提示词

skills.ts:94-116
formatSkillsForSystemPrompt(): string {
  // 只包含 disableModelInvocation !== true 的 skill
  const visible = this.skills.filter(s => !s.disableModelInvocation);

  return [
    "The following skills provide specialized instructions for specific tasks.",
    "Read the full skill file when the task matches its description.",
    "",
    "<available_skills>",
    for (const skill of visible) {
      `  <skill>`,
      `    <name>${skill.name}</name>`,
      `    <description>${skill.description}</description>`,
      `    <location>${skill.location}</location>`,
      `  </skill>`,
    },
    "</available_skills>",
  ].join("\n");
}

3.3 按需读取全量内容

skills.ts:119-125
async readSkillContent(name: string): Promise<string> {
  const skill = this.getSkill(name);
  if (!skill) throw new Error(`Skill "${name}" not found.`);
  return fs.readFile(skill.location, "utf-8");
}

注意：description 是预加载的（扫描时解析），正文是 lazy 加载的——只在 LLM 决定使用某个 skill 时才读取完整文件。

4. Skill 在对话中的生命周期

 一次 Skill 激活的完整流程

① 启动 SkillManager.loadSkills() → 扫描所有 SKILL.md → 提取 description
② 拼装 formatSkillsForSystemPrompt() → XML 片段 → 加入系统提示词
③ 对话 LLM 看到 <available_skills>，发现 "tdd" 匹配用户请求
④ 读取 LLM 使用 read 工具 → readSkillContent("tdd") → 完整 SKILL.md 内容
⑤ 注入 SKILL.md 内容作为消息加入对话 → LLM 按 skill 指令执行
⑥ 执行 LLM 可能输出回答或 tool_calls → 进入 ReAct 循环（第 2 课）

5. Skill vs Tool

很多初学者容易混淆 Skill 和 Tool，它们的区别非常明确：

维度 Tool（工具） Skill（技能）
本质 可执行代码 文本指令（prompt）
注册方式 toolRegistry.register() 放在 skills/ 目录即可
LLM 感知方式 通过 tools API 参数 嵌入系统提示词
作用时机 LLM 返回 tool_calls 时 贯穿整个对话（指导 LLM 行为）
扩展方式 写 TypeScript 代码 写 Markdown 文件

6. disable-model-invocation 的作用

有些 skill 不打算让 LLM 自动发现（比如 teach）。它们设置 disable-model-invocation: true：

skills.ts:94-96
// ⭐ 过滤掉 disableModelInvocation 的 skill
const visible = this.skills.filter(s => !s.disableModelInvocation);

这类 skill 不会出现在系统提示词的 <available_skills> 中，LLM 不知道它们的存在。它们只能通过 slash 命令（如 /teach、/code-review）由用户主动触发。

💡 两种 skill 场景：
- 自动 skill（disableModelInvocation: false）→ LLM 自动判断何时使用
- 命令 skill（disableModelInvocation: true）→ 用户通过 /command 主动调用
