# 0003 - Agent 交互日志记录

## 状态

已接受

## 上下文

当前项目没有文件日志记录功能，所有输出仅通过控制台显示。用户需要记录大模型输出和 agent 交互到文件，以便于调试、分析和审计。

## 决策

设计一个轻量级、高性能的日志模块，主要特点：

1. **日志存储**: `~/.babyAgent/logs/` 目录下，按日期和会话 ID 分文件
2. **日志内容**: 记录 LLM 请求/响应、工具调用、会话事件、错误信息
3. **性能**: 异步写入，不阻塞主流程
4. **配置**: 支持环境变量配置日志级别和目录

## 设计方案

### 1. 日志目录结构
```
~/.babyAgent/logs/
├── 2026-07-05/
│   ├── session-{sessionId}.log
│   └── ...
└── 2026-07-04/
    └── ...
```

### 2. 日志格式
```json
{
  "timestamp": "2026-07-05T14:30:00.000Z",
  "level": "INFO",
  "component": "agent",
  "event": "llm_request",
  "sessionId": "abc123",
  "data": {
    "model": "deepseek-chat",
    "messages": [...]
  }
}
```

### 3. 日志级别
- `ERROR`: 错误和异常
- `WARN`: 警告信息
- `INFO`: 关键事件（LLM 调用、工具执行、会话事件）
- `DEBUG`: 详细调试信息

### 4. 核心组件
- `Logger`: 单例模式，统一日志管理
- `FileTransport`: 异步文件写入
- `LogConfig`: 配置管理

## 后果

### 优点
- 统一的日志管理
- 异步写入不阻塞主流程
- 结构化日志便于分析
- 按日期和会话分类便于查找

### 缺点
- 增加文件 I/O 开销
- 需要管理磁盘空间
- 日志文件可能累积

## 相关文件
- `src/agent.ts` - Agent 类，需要记录 LLM 交互
- `src/coordinator.ts` - Coordinator 类，需要记录会话事件
